Credit Card Fraud Detection in R

This project uses machine learning in R to identify fraudulent credit card transactions from an anonymized dataset. The goal is to build and evaluate classification models that can distinguish fraud from legitimate purchases, while accounting for the strong class imbalance that typically exists in fraud detection problems.

The analysis includes data preprocessing, stratified sampling, cross-validation, model training, and performance evaluation. Because fraud cases are rare compared to normal transactions, the project focuses not just on overall accuracy, but on metrics that better reflect model quality in imbalanced classification settings.

Project Goals:
Build a predictive model for credit card fraud detection
Preprocess and prepare transaction data for modeling
Compare model performance using appropriate evaluation metrics
Understand the challenges of detecting rare fraud cases in real-world data

Tools Used:
- R
- tidyverse
- tidymodels
- ggplot2
- R Markdown
  
What the Project Covers:

This project walks through the full workflow of a classification problem in R, including:

data cleaning and preparation
train/test splitting
cross-validation
model fitting
model evaluation and interpretation
Why This Project Matters

Credit card fraud detection is a real-world machine learning problem with direct business relevance. It highlights the importance of predictive modeling, careful evaluation, and handling imbalanced datasets in a way that reflects practical decision-making.
